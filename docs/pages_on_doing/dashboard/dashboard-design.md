# 2026-07-11 11:15:08 - 대시보드 작업 시작
대시보드 페이지는 사용자가 `index.html`에서 로그인 후 토큰과 함께 리다이렉트 되는 공간입니다. 

들어왔을 시 실행되는 작업의 경우에는

1. `/auth/me`를 통한 인증 절차 (안되면 `/auth/refresh` 한번 시도할 수 있음)
2. 사용자의 정보를 불러와 정보들 보여주며 필요에 따라 기록, 알람, 로그아웃 기능까지 제공하기 (로그아웃은 설계에 없지만 구현이 어렵지 않으니)
3. 연결 성공 시 `socket` 연결
4. `p2p` 또는 `room` 페이지로 이동/생성 가능 (이때 url을 통해서 직접 접속하는 경우를 산정하여 비번은 방 안에서 검수 후 socket/transport consume 등 작업 시작)

> 중요한 점은 SPA의 경우에는 다르겠지만 현재 방식은 url별로 서버에 요청하여 자원(스크립트)를 가져오는 방식이기 때문에 새로고침 시에도 페이지 초기 로딩과 동일하게 생각이 가능하며 통화방 이탈 시의 이벤트는 서버쪽에서 다루면 될것이라 판단하였습니다. 또한 비밀번호 입력은 링크를 직접 입력하는 경우를 산정하여 `p2p`, `room`에서 작업하려 합니다.

여기에서 작업할 기능들의 존재 이유를 하나씩 살펴보면

1. `/me`와 `/refresh`: 다음 모든 작업의 재개 여부를 다룸. 사용자가 js를 조작할 수 있기 때문에 실제로 어거지로 다음 작업을 실행해도 서버의 미들웨어에서 막히게 할 예정
2. `socket connect`: 사실 주요 기능은 아니지만 초대를 받게 하기 위해선 `.on("invite")`와 같은 작업을 위한 연결입니다. 어짜피 `socketio`의 주체는 브라우저 탭이 아닌 `js` 스크립트의 `Socket`객체이기 때문에 여기에서 연결이 되건 안되건 다른 페이지에서 연결을 다시 해야합니다.
3. 버튼으로는 `로그아웃`, `p2p방 생성` `p2p방 입장` `회의 참여(group)` `회의 생성(group)` 이 존재합니다. 이후 추가적으로 `activity_log`를 이용하여 내 기록 보기 -> `lazy fetch` 방식을 통해 기록 보여주기 기능을 제공할 생각입니다.
4. (선택) 방 안에서 미디어 트랙 작업을 지연시키지 않기 위해 자신의 미디어를 서버의 `codec`와 미리 대조 가능하게 하는 기능을 제작할 생각입니다.

## 사용할 이벤트/소켓/작업 등
이제 진짜 코드적으로 할 작업들을 정의할 생각입니다.

코드 플로우
1. `/auth/me` 를 통해 로그인 확인 
  - 실패하면 `/auth/refresh` 작업
    - 성공하면 다시 `/auth/me` 작업 후 2번으로
    - 이것도 실패하면 알람 후 `index.html`로 리다이렉트
2. 사용자 정보 먼저 보여주기 (`/auth/me` 결과로)
3. 사용자에게 `name`, `email`, `img src=profile_image_link` 보여주기.
4. `socket` 서버와 연결하기 
5. `socket.on("invited:received")` 이벤트 추가해주기 (이벤트 발생 시 사용자 우측 상단에 방 입장 버튼 링크 만들어주기)
6. 미디어 확인 버튼으로 서버에 `GET /rooms/:roomCode/rtp-capabilities`를 보내어 사용 가능한 미디어 확인할 수 있게 해주기 (그런데 이건 방 안에서 하는게 맞는듯. 대시보드에서는 medialist만 보여주기)
7. 방 생성 누르면 비번 자동생성 및 사용자를 `/room.html?roomCode=...` 또는 `/p2p.html?roomCode=...`으로 보내주기 (닉네임은 안쪽에서 설정. 기본 닉네임 제공해주기. 외부에는 닉네임이 보이고 자세히 보면 이메일과 이름 확인 가능하게 하기)
8. 방 입장하기 누르면 방 코드 입력받아주고 보내주거나 없다고 알려주기
