# 서비스 플로우가 어떻게 흐르는지

현재 프로젝트에는 4개의 페이지를 축으로 4가지 주요 기술이 사용되었습니다.
- Google Oauth 기반의 사용자 인증흐름 (+jwt)
- 웹소켓 기반 이벤트 기반 프로그래밍
- RTCPeerConnection을 통한 P2P 통화방 시그널링 서버
- mediasoup 기반의 SFU 서버 및 방 관리
> 추가로 서버 메모리 기반의 사용자/방 상태관리

## `index.html`
해당 서버는 기본적으로 SSG 렌더링 방식을 통해 `.html`을 정적 자원 제공을 통해 응답해준뒤 `.html` 내부의 `<script>`를 통해 서버의 `.js` 파일로 동적으로 연결이 되는 방식을 사용하였습니다.

### `index.html`의 역할
`index.html`은 기본적으로 `/` 에서도 응답이 나가 로그인 되어있지 않은 사용자들이 로그인을 하는 공간으로 제작하였습니다.

해당 페이지를 들어가면 최초로 `loadMe()` 작업을 하여 이미 토큰이 존재하는지 확인 후 로그인 되어있을 시 `dashboard.html`로 이동시키며 로그인 되어있지 않은 경우에는 페이지를 그대로 보여주어 로그인을 할 수 있게 해줍니다.

## `dashboard.html`
대시보드는 로그인 된 사용자가 실제 서비스 사용을 할 수 있는 페이지입니다.
